# AI技術ブログ記事ジェネレーター

AIが技術ブログ記事のテーマ提案から執筆までをサポートするWebアプリケーションです。AIのペルソナ、参考環境、Markdownフォーマットを自由に設定でき、あらゆるブログプラットフォームに対応可能です。

## 主な機能

- **AIによるテーマ提案**: 開発者が直面する技術的な課題やエラー解決策をテーマに、ユニークで関心を引く記事テーマを複数提案します。
- **柔軟なAI設定**:
  - **ペルソナ**: AIの専門性や記事のトーン（例: 備忘録スタイル、丁寧な解説スタイル）を自由に指示できます。
  - **参考環境**: 記事で言及する開発環境（OS、ツール、バージョン）を設定できます。
  - **フォーマット**: 投稿先のプラットフォームに合わせてMarkdownのルール（Front Matter、記事構成など）をAIに指示できます。
- **カスタマイズ可能なMarkdown生成**: 設定したルールに基づき、指定したブログプラットフォームのフォーマットに準拠した高品質なMarkdown記事を自動生成します。
- **履歴機能**: 過去に生成したテーマを自動で保存し、重複しない新しいテーマを提案することで、アイデアのマンネリ化を防ぎます。
- **簡単な操作**: 直感的なUIで、ボタンをクリックするだけで誰でも簡単に記事を作成できます。

## 技術スタック

- **フロントエンド**: React, TypeScript, Tailwind CSS
- **AI**: Google Gemini API (@google/genai)
- **開発サーバー**: serve

## APIキーの設定

このアプリケーションを利用するには、Google Gemini APIのAPIキーが必要です。

1.  [Google AI Studio](https://aistudio.google.com/app/apikey) にアクセスしてAPIキーを取得します。
2.  アプリケーション右上の設定アイコン（歯車マーク）をクリックします。
3.  表示された設定パネルの「Gemini APIキー」入力フィールドに、取得したキーを貼り付けて保存します。

APIキーはブラウザの`localStorage`に保存され、外部に送信されることはありません。

## ローカルでの実行方法

1.  **リポジトリをクローンします:**
    ```bash
    git clone <repository-url>
    cd <repository-directory>
    ```

2.  **依存関係をインストールします:**
    ```bash
    npm install
    ```

3.  **開発サーバーを起動します:**
    ```bash
    npm run dev
    ```
    ブラウザで `http://localhost:3000` を開くと、アプリケーションが表示されます。その後、上記の「APIキーの設定」の手順に従ってキーを設定してください。(`npm start`も利用可能です。)

## デプロイ

このアプリケーションは、Render, Vercel, Netlifyのような静的ホスティングサービスに簡単にデプロイできます。環境変数を設定する必要はありません。

### Renderでのデプロイ手順

このリポジトリには `render.yaml` ファイルが含まれており、Renderの**Blueprint**機能を使って簡単にデプロイできます。

1.  [Render](https://render.com/) にサインアップし、GitHubアカウントを接続します。
2.  ダッシュボードから **[New +]** > **[Blueprint]** をクリックします。
3.  デプロイしたいGitHubリポジトリを選択し、**[Connect]** をクリックします。
4.  Renderが自動的に `render.yaml` を読み込み、サービス名（例: `ai-tech-blog-generator`）を提案します。
5.  **[Apply]** をクリックしてデプロイを開始します。

`render.yaml` ファイルがビルドコマンドや公開ディレクトリ、さらに本番環境で`.tsx`ファイルを正しく読み込むためのサーバー設定を自動で行うため、手動での設定は不要です。

### Vercel / Netlify でのデプロイ

VercelやNetlifyでは、`.tsx`ファイルを正しく配信するためにカスタムヘッダーの設定が必要になる場合があります。

1.  **Gitリポジトリにプッシュします。**
2.  **ホスティングサービスを選択します:**
    - Vercel, Netlify など、お好みのサービスにリポジトリを接続します。
3.  **ビルド設定を構成します:**
    - **Build Command**: `npm install`
    - **Publish Directory / Output Directory**: `.` (ルートディレクトリ)
4.  **カスタムヘッダーを設定します (必要な場合):**
    - `vercel.json` や `netlify.toml` を作成し、`.tsx` と `.ts` ファイルの `Content-Type` ヘッダーを `text/javascript` に設定するルールを追加します。
5.  **デプロイを実行します。**

デプロイ後、アプリケーションにアクセスしたユーザーは、各自のAPIキーを設定画面から入力して利用します。
