# AI技術ブログ記事ジェネレーター

AIがZenn向けの技術ブログ記事のテーマ提案から執筆までをサポートするWebアプリケーションです。開発者が直面する技術的な課題やエラー解決策をテーマにした高品質な記事を、簡単な操作で生成できます。

## 主な機能

- **AIによるテーマ提案**: 最新のトレンドや開発者のニーズに基づき、ユニークで関心を引く記事テーマを複数提案します。
- **Zenn向けMarkdown生成**: 提案されたテーマに基づき、Zennのフォーマット（Front Matter、メッセージボックス等）に完全に準拠したMarkdown形式の記事を自動生成します。
- **履歴機能**: 過去に生成した記事のテーマを自動で保存し、重複しない新しいテーマを提案することで、アイデアのマンネリ化を防ぎます。
- **簡単な操作**: 直感的なUIで、ボタンをクリックするだけで誰でも簡単に記事を作成できます。

## 技術スタック

- **フロントエンド**: React, TypeScript, Tailwind CSS
- **AI**: Google Gemini API (@google/genai)
- **開発サーバー**: serve

## 環境変数

このアプリケーションを実行するには、Google Gemini APIのAPIキーが必要です。

プロジェクトのルートディレクトリに `.env` ファイルを作成し、以下のようにAPIキーを設定してください。

```
API_KEY="YOUR_GEMINI_API_KEY"
```

`YOUR_GEMINI_API_KEY` は、[Google AI Studio](https://aistudio.google.com/app/apikey)で取得した実際のキーに置き換えてください。

## ローカルでの実行方法

1.  **リポジトリをクローンします:**
    ```bash
    git clone <repository-url>
    cd <repository-directory>
    ```

2.  **依存関係をインストールします:**
    ```bash
    npm install
    ```

3.  **環境変数を設定します:**
    上記の「環境変数」セクションに従い、`.env` ファイルを作成してAPIキーを設定してください。

4.  **開発サーバーを起動します:**
    ```bash
    npm start
    ```
    ブラウザで `http://localhost:3000` を開くと、アプリケーションが表示されます。

    **注意**: このプロジェクトはビルドツールを使用していないため、`process.env.API_KEY` はローカルサーバーでは直接利用できません。デプロイ環境での動作を前提としていますが、ローカルで試す場合は、`index.html` に直接APIキーを書き込むなどの一時的な対応が必要です。

## デプロイ

このアプリケーションは、VercelやNetlifyのような静的ホスティングサービスに簡単にデプロイできます。

1.  **Gitリポジトリにプッシュします。**
2.  **ホスティングサービスを選択します:**
    - Vercel, Netlify, GitHub Pages など、お好みのサービスにリポジトリを接続します。
3.  **ビルド設定を構成します:**
    - このプロジェクトはビルドステップを必要としないため、ビルドコマンドは空のままで構いません。
    - 公開ディレクトリはルートディレクトリ（`.` または `dist` などサービスが指定するもの）に設定します。
4.  **環境変数を設定します:**
    - ホスティングサービスの管理画面で、`API_KEY` という名前の環境変数を設定し、値にあなたのGemini APIキーをセットします。
    - **重要**: 多くのホスティングサービス（Vercelなど）では、フロントエンドで環境変数を利用可能にするために、変数名にプレフィックス（例: `VITE_` や `NEXT_PUBLIC_`）が必要です。このプロジェクトはビルドツールを使わないため、お使いのホスティングサービスが環境変数をどのように静的ファイルに注入するか、ドキュメントをご確認ください。

5.  **デプロイを実行します。**
