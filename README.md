
# AI技術ブログ記事ジェネレーター

AIが技術ブログ記事のテーマ提案から執筆までをサポートするWebアプリケーションです。AIのペルソナ、参考環境、Markdownフォーマットを自由に設定でき、あらゆるブログプラットフォームに対応可能です。

## 主な機能

- **AIによるテーマ提案**: 開発者が直面する技術的な課題やエラー解決策をテーマに、ユニークで関心を引く記事テーマを複数提案します。
- **柔軟なAI設定**:
  - **ペルソナ**: AIの専門性や記事のトーン（例: 備忘録スタイル、丁寧な解説スタイル）を自由に指示できます。
  - **参考環境**: 記事で言及する開発環境（OS、ツール、バージョン）を設定できます。
  - **フォーマット**: 投稿先のプラットフォームに合わせてMarkdownのルール（Front Matter、記事構成など）をAIに指示できます。
- **カスタマイズ可能なMarkdown生成**: 設定したルールに基づき、指定したブログプラットフォームのフォーマットに準拠した高品質なMarkdown記事を自動生成します。
- **履歴機能**: 過去に生成したテーマを自動で保存し、重複しない新しいテーマを提案することで、アイデアのマンネリ化を防ぎます。
- **簡単な操作**: 直感的なUIで、ボタンをクリックするだけで誰でも簡単に記事を作成できます。

## 技術スタック

- **フロントエンド**: React, TypeScript, Tailwind CSS
- **AI**: Google Gemini API (@google/genai)
- **開発サーバー**: serve

## APIキーの設定

このアプリケーションを利用するには、Google Gemini APIのAPIキーが必要です。

1.  [Google AI Studio](https://aistudio.google.com/app/apikey) にアクセスしてAPIキーを取得します。
2.  アプリケーション右上の設定アイコン（歯車マーク）をクリックします。
3.  表示された設定パネルの「Gemini APIキー」入力フィールドに、取得したキーを貼り付けて保存します。

APIキーはブラウザの`localStorage`に保存され、外部に送信されることはありません。

## ローカルでの実行方法

1.  **リポジトリをクローンします:**
    ```bash
    git clone <repository-url>
    cd <repository-directory>
    ```

2.  **依存関係をインストールします:**
    ```bash
    npm install
    ```

3.  **開発サーバーを起動します:**
    ```bash
    npm start
    ```
    ブラウザで `http://localhost:3000` を開くと、アプリケーションが表示されます。その後、上記の「APIキーの設定」の手順に従ってキーを設定してください。

## デプロイ

このアプリケーションは、VercelやNetlifyのような静的ホスティングサービスに簡単にデプロイできます。環境変数を設定する必要はありません。

1.  **Gitリポジトリにプッシュします。**
2.  **ホスティングサービスを選択します:**
    - Vercel, Netlify, GitHub Pages など、お好みのサービスにリポジトリを接続します。
3.  **ビルド設定を構成します:**
    - このプロジェクトはビルドステップを必要としないため、ビルドコマンドは空のままで構いません。
    - 公開ディレクトリはプロジェクトのルートディレクトリに設定します。
4.  **デプロイを実行します。**

デプロイ後、アプリケーションにアクセスしたユーザーは、各自のAPIキーを設定画面から入力して利用します。
